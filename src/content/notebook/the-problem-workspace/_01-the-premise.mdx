The file system is not a filing cabinet. It's a protocol.

Where you put something determines whether an agent can use it. The question is no longer "where do I file this?" but "how do I package knowledge so an agent can act on it?" Folder structure is the interface. Get it wrong, and even the best model suffocates on noise.

> **Definition:** A [[problem-workspace]] is a directory structure designed for human–AI collaboration, not human archiving.

Most people still dump everything into a chat window and hope. **~10,000 tokens** of raw text, a few PDFs, maybe a screenshot. The model nods along. Nothing good happens. Context windows are wide now, but width is not depth. Unstructured input produces unstructured output. The workspace is the corrective.

Treat the folder as the collaboration interface. You and the agent read from the same tree. When the structure is explicit, the agent knows where it is in the process. When it is not, you are both lost.
