This is where things get built. Finally.

Solutions live in `iterations/`, not in root, not in drafts. The naming convention matters. Drafts are temporary. Iterations are committed attempts.

```
iterations/
├── iteration-01/
│   ├── design/
│   ├── prototype/
│   └── analysis.md
└── iteration-02/
```

Each iteration contains what was tried, what was learned, and what failed. The folder holds designs, code, models, or documents. The `analysis.md` file records the post-mortem before you move on.

The language is intentional. A draft is cheap. An iteration costs attention. When you name something `iteration-03`, you're making a claim. You're saying this was a real attempt with real stakes.

The AI generates. The human reviews. No review, no iteration. The output of this layer feeds directly into [[Review]] — which is where you decide if the frame was right, if the context was enough, or if the work solves anything at all.
